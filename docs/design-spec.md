# Learning Consultant System Design

> 타협하지 않는 학습 컨설턴트 — 근거 기반 방법론으로 학습 방향을 교정하는 시스템

## Problem Statement

- 하루 10시간 공부하지만 결과가 노력에 비례하지 않음
- 자료 만들기(정리, 노트시트)에 시간이 편중 → 수동적 처리(passive processing)
- 개념은 아는 것 같은데 응용 문제에서 막힘 → 전이(transfer) 실패
- 근본 원인: 유창성 착각(illusion of fluency) + 메타인지 부족

## Core Philosophy

### 학습과학 원칙 (non-negotiable)

1. **인출 연습 (Retrieval Practice)** — 읽기/정리 = 재인식(recognition). 시험은 인출(recall)을 요구. 백지에서 쓸 수 있어야 "안다"고 할 수 있음. 학습 효과 2-3배 차이.
2. **바람직한 어려움 (Desirable Difficulty)** — 공부가 쉽게 느껴지면 학습이 안 되고 있을 가능성. 자료 정리가 위험한 이유 — "생산적인 느낌"이지만 인지적으로 얕은 처리.
3. **전이 적절 처리 (Transfer-Appropriate Processing)** — 시험이 요구하는 인지 과정과 공부 시 인지 과정이 일치해야 함. 응용 문제를 내면, 응용을 연습해야 함.

### 컨설턴트 태도

- 친절하지만 양보하지 않음
- "이거 알아"를 액면 그대로 받지 않음 — 항상 증거 요구
- 편한 방법 선택 시 왜 비효율적인지 설명 + 교정
- 인신공격/모욕 절대 없음 — 방법론에 대한 엄격함이지 사람에 대한 엄격함이 아님

### 적극적 개입 메커니즘 (6가지)

1. **세션 시작 자동 체크인** — 매 세션 시작 시 계획/복습 큐/리뷰 상태 확인
2. **일정 역산** — Google Calendar 또는 CLAUDE.md/goals.md의 시험 일정 기반으로 자동 우선순위 조정 (캘린더 접근 불가 시 CLAUDE.md fallback)
3. **간격 반복 스케줄러** — study-log 기반 자동 복습 알림
4. **자료 생성 게이트** — Type A~D 호출 시 선행 조건 강제
5. **주간 리뷰 자동 트리거** — 7일 이상 리뷰 안 했으면 알림
6. **계획 부재 감지** — 현재 계획 없거나 만료 시 "계획부터 세우자" 제안

## System Architecture

```
┌─────────────────────────────────────────────┐
│              학습 컨설턴트 시스템              │
├─────────────────────────────────────────────┤
│                                             │
│  ┌───────────┐   자동 개입                   │
│  │  Rules    │──→ R1: 세션 시작 체크인          │
│  │(.claude/  │──→ R2: 자료 생성 게이트         │
│  │ rules/)   │──→ R3: 학습 활동 적절성 판단     │
│  │           │──→ R4: 주간 리뷰 트리거         │
│  │           │──→ R5: 계획 부재 감지           │
│  └───────────┘                              │
│                                             │
│  ┌───────────┐   사용자 호출                  │
│  │  Skills   │                              │
│  │(.claude/  │──→ /study  (상시 컨설턴트)     │
│  │ skills/)  │──→ /diagnose (이해도 진단)     │
│  │           │──→ /plan (학습 계획 수립)       │
│  │           │──→ /review (주간 리뷰)         │
│  └───────────┘                              │
│                                             │
│  ┌───────────┐   데이터                      │
│  │  State    │                              │
│  │           │──→ goals.md (목표 계층)        │
│  │           │──→ study-plan.md (주간 계획)   │
│  │           │──→ spaced-queue.md (복습 큐)   │
│  │           │──→ study-log/ (일별 기록)      │
│  │           │──→ mastery.md (주제별 이해도)   │
│  │           │──→ feedback/ (주간+월간 피드백) │
│  └───────────┘                              │
│                                             │
│  ┌───────────┐   외부 연동                    │
│  │  MCP      │──→ Google Calendar (일정 역산)  │
│  │           │   (불가 시 CLAUDE.md fallback) │
│  └───────────┘                              │
│                                             │
├─────────────────────────────────────────────┤
│  기존 도구 (Type A~D)                        │
│  → Rules 게이트를 통과해야만 사용 가능          │
└─────────────────────────────────────────────┘
```

**레이어 역할:**
- **Rules** — 자동 가드레일. 호출 없이 작동.
- **Skills** — 명시적 호출 기능. 같은 State 공유.
- **State** — 마크다운 플랫 파일. DB 없음. 사람이 직접 읽을 수 있음.
- **MCP** — 캘린더에서 시험/수업 일정을 가져와 자동 우선순위 조정. 캘린더 접근 불가 시 CLAUDE.md + goals.md의 일정 정보를 fallback으로 사용.

## Skills Detail

### `/study` — 상시 컨설턴트 모드

세션 관리자가 아닌 **공부 도중 함께하는 대화 상대.**

**시작 시:**
1. `study-plan.md` 확인 → 없거나 만료면 `/plan`으로 리다이렉트
2. `spaced-queue.md` 확인 → 오늘 복습할 항목 최우선 제시
3. 시험/과제 일정 확인 (Google Calendar → 불가 시 CLAUDE.md/goals.md fallback) → 긴급도 반영
4. "오늘의 학습 블록" 제안

**도중:**
- 실시간으로 학습 활동 적절성 판단 (Rule 3 프레임워크)
- "지금 그거 하지 마, 이거 해" 식의 교정
- 자연스러운 진단 통합 ("이거 안다고? 설명해봐")
- 자료 생성 요청 시 게이트 체크

**종료 시:**
1. 자기 보고 수집 ("오늘 뭘 했어?")
2. 시간 배분 분석 (자료 제작 vs 인출 vs 문제 풀이 비율)
3. 자료 제작 비율 높으면 경고 + 교정 제안
4. `study-log/` 기록, `spaced-queue.md` 업데이트

### `/diagnose` — 독립 이해도 진단

세션 밖에서 특정 주제만 빠르게 테스트.

**상태 업데이트:** `/diagnose` 실행 후 결과를 `mastery.md`와 `spaced-queue.md` 양쪽에 반영. mastery.md가 이해도의 단일 원천(single source of truth)이고, spaced-queue.md는 mastery 변경에 따라 간격을 재계산.

**3단계 진단:**
- **Level 1 — 인출:** "핵심 개념 3가지를 아무것도 안 보고 설명해봐"
- **Level 2 — 연결:** "이 개념이 [이전 주제]와 어떻게 연결돼?"
- **Level 3 — 응용:** 변형 문제 제시, 풀이 요구

**레벨별 처방:**
- L1 실패: 기본 개념 재학습 (자료 참고 허용)
- L2 실패: 개념 간 관계 매핑 연습
- L3 실패: 유사 문제 반복 + 오답 분석

**타협 불가:** "대충 맞았으니 넘어가자" 없음. 기준 못 넘기면 다음 주제로 못 감.

### `/plan` — 학습 계획 수립

**프로세스:**
1. 수강 과목, 시험 일정 파악 (Google Calendar → 불가 시 CLAUDE.md/goals.md fallback)
2. 각 과목 ToC 참조 → 남은 범위 vs 남은 시간 계산
3. 주 단위 계획 수립: 과목별 시간 배분, 주제 우선순위, 인출/문제풀이 비율 강제
4. `study-plan.md`에 저장

**타협 불가:** "이건 나중에" → 일정 역산 근거로 판단. 감이 아닌 데이터로 결정.

### `/review` — 주간 리뷰

**프로세스:**
1. `study-log/`에서 이번 주 전체 분석
2. `goals.md`의 상위 목표 대비 진행도 체크
3. 리포트: 총 시간, 과목별 배분, 활동 비율, 간격 반복 준수율, 진단 결과 요약
4. `feedback/YYYY-MM.md`에 주간 피드백 기록
5. 다음 주 계획 조정 제안

**월말:** 해당 월 주간 피드백 종합 → 월간 종합 피드백 추가.

## Rules — 자동 개입 레이어

### Rule 1: 세션 시작 체크인
- **트리거:** 매 세션 시작
- **동작:** study-plan 존재 여부 / spaced-queue 오늘 항목 / 최근 review 날짜 확인 → 알림

### Rule 2: 자료 생성 게이트
- **트리거:** Type A~D 스킬 호출 시
- **동작:** Type별 허용 조건 체크:

```
Type A (종합 자료):
  → mastery 단계 확인
  → 첫 노출 단계 = 허용 (한국어 이해용)
  → 한국어 정착 이상 + 인출 기록 없음 = 먼저 진단 요구
  → 인출 후 빈 곳 보강 = 허용 (목적 있는 참조)

Type B (노트시트):
  → mastery L2 이상 + 영어 전환 완료 = 허용
  → 미충족 = 차단 ("아직 L2 안 됐어. 먼저 인출 연습해")

Type C (Anki):
  → 진단 실패 개념이 있는 경우 = 허용 (약한 개념 타겟)
  → 진단 없이 "전체 카드 만들기" = 차단 ("먼저 뭐가 약한지 진단해")

Type D (문제집):
  → 항상 허용 (문제 풀이는 어떤 단계에서든 유익)
```

### Rule 3: 학습 활동 적절성 판단

키워드 매칭이 아닌 **프레임워크 기반 판단:**

```
Step 1: 학습 단계 파악
  → study-log/에서 이 주제의 이력 확인
  → 첫 노출 / 한국어 정착 / 영어 전환 / 응용 / 시험 준비

Step 2: 제안된 활동의 인지 수준 분류
  → 재인식(읽기, 정리) / 인출(백지 테스트) / 응용(문제 풀이)

Step 3: 단계-활동 매칭 판정
  → 첫 노출 + 읽기 = 허용
  → 한국어 정착 + 읽기 = 개입 ("한국어 인출 먼저")
  → 영어 전환 + 한국어만 = 개입 ("영어로 전환해")
  → 시험 준비 + 정리 = 개입 ("문제 풀어")
  → 인출 후 빈 곳 + 읽기 = 허용 (목적 있는 참조)

Step 4: 개입 시 → 이유 한 줄 + 대안 제시
```

**로그 없는 초기:** "이 주제 처음이야?" 물어본 뒤 답변 기반 판단.

### Rule 4: 주간 리뷰 트리거
- **트리거:** 매 세션 시작 시 (Rule 1과 함께 실행)
- **동작:** `feedback/` 디렉토리에서 마지막 주간 피드백 날짜 확인 → 7일 초과 시 "주간 리뷰 할 시간이야. `/review` 먼저 하자" 알림. 단, 사용자가 긴급한 작업 중이면 세션 종료 시 다시 알림.

### Rule 5: 계획 부재 감지
- **트리거:** 매 세션 시작 시 (Rule 1과 함께 실행)
- **동작:** `study-plan.md` 없음 또는 `expires` 날짜 지남 → "계획이 없거나 만료됐어. 공부 시작 전에 `/plan`부터 하자" 제안. 계획 없이 학습 진행하는 것을 허용하지 않음.
- **Cold Start 우선:** `goals.md`, `mastery.md` 중 하나라도 없으면 Rule 5 대신 Cold Start 프로세스를 따른다. Rule 5는 Cold Start 완료 후(3개 기초 파일 모두 존재) 에만 적용.

## State — 데이터 구조

모든 파일은 마크다운. 단순하고 사람이 읽을 수 있음.

### `goals.md` — 목표 계층

```markdown
# 연간 목표 (2025-2026 Academic Year)
- ...

# W26 쿼터 목표 (2026-01 ~ 2026-03)
- CSE152A: A- 이상
- CSE101: B+ 이상
- CSE100: A- 이상

# 3월 목표
- CSE152A: 전 범위 L2 이상, Final 응용 문제 사이클 완료
- CSE101: DP 약점 해소, Final 인출 1사이클
- CSE100: 전 범위 첫 노출 완료
```

### `study-plan.md` — 주간 계획

```markdown
---
created: YYYY-MM-DD
expires: YYYY-MM-DD
---
## 주간 목표
## 시간 배분
## 이번 주 원칙
```

### `spaced-queue.md` — 간격 반복 큐

```markdown
## 오늘 복습 (YYYY-MM-DD)
- [ ] 주제 (간격, 마지막 학습일, 진단 결과)

## 내일
## 이번 주 예정
```

### `study-log/YYYY-MM-DD.md` — 일별 학습 기록

```markdown
# YYYY-MM-DD
## 세션 N (시작-종료)
- 과목 / 활동 / 비율 / 진단 결과
```

### `mastery.md` — 주제별 이해도

```markdown
## 과목명
| 주제 | 단계 | 언어 | 최근 진단 | 결과 | 다음 액션 |
```

단계: 첫 노출 / 한국어 정착 / 영어 전환 / 응용 / 시험 준비
언어: - / 한국어 / 영어

### `feedback/YYYY-MM.md` — 주간 + 월간 피드백 통합

```markdown
# W10 주간 피드백 (2026-03-07)
## 잘한 점
## 아쉬운 점
## 패턴 감지
## 다음 주 핵심 교정

# W11 주간 피드백 (2026-03-14)
...

# 3월 종합 피드백
## 이번 달 성장
## 반복된 아쉬운 점
## 메타인지 인사이트
## 다음 달 전략 조정
```

**피드백 검증 계층:**
```
연간 목표 ← 쿼터 종료 피드백이 검증
  쿼터 목표 ← 월간 피드백이 검증
    월간 목표 ← 주간 피드백이 검증
      주간 계획 ← 일별 로그가 검증
```

## 기존 도구 통합 — Type A~D 리포지셔닝

### 학습 단계 전체 사이클 (언어 전환 포함)

```
1. 첫 노출
   수업(영어) → Type A 한국어 자료로 이해
   허용: 읽기, 자료 생성

2. 한국어 정착
   한국어로 인출 테스트 → 빈 곳 Type A 참조
   필수: 한국어 인출 통과해야 다음 단계

3. 영어 전환
   영어로 인출 테스트 → 용어/설명 영어로 전환
   필수: 영어 인출 통과해야 다음 단계

4. 응용
   Type D 문제 풀이(영어) → 오답 분석
   Type C Anki 카드(약한 개념만)

5. 시험 준비
   영어 인출 + 문제 풀이만
   Type B 노트시트(영어, L2 이상만)
```

### Type별 새로운 위치

| Type | 새 역할 | 허용 조건 |
|------|--------|----------|
| A (종합 자료) | 첫 노출 시 한국어 이해용 | 첫 노출 단계에서 허용 |
| B (노트시트) | 시험 직전 영어 압축 | mastery L2 이상 + 영어 전환 완료 |
| C (Anki) | 약한 개념 타겟 카드 | 진단 실패 개념 위주, 영어 |
| D (문제집) | 학습 사이클 핵심 | 가장 자유롭게 허용 (영어) |

## 보완 사항

### Crunch Mode — 시간 압박 시 사이클 압축

```
D-7 이상:  정상 사이클

D-3~6:    압축 사이클
  → Type A 한국어 이해 + 즉시 한국어 인출 (같은 세션)
  → 바로 영어 전환 + Type D 문제 풀이
  → 자료 정돈 금지

D-1~2:    서바이벌 모드
  → 새 주제 학습 중단. 기존 주제 강화.
  → L1 이상만 영어 인출 + 문제 풀이
  → L1 미달은 Type B에 공식만 넣고 참조용
```

Crunch Mode에서도 "읽기만 하기"는 차단. 시간이 없을수록 인출이 더 중요.

### Cold Start — 초기 셋업

State 파일이 하나도 없는 상태에서 **첫 `/study` 또는 `/plan` 호출 시** 자동으로 초기 셋업 모드 진입. `/study`의 정상 시작 로직(plan 체크 등)보다 Cold Start가 우선.

**프로세스:**
1. 기본 정보 수집 (과목, 일정, 진도)
2. 과목별 자가 진단 ("자신 있는 것 / 불안한 것")
3. 목표 설정 → `goals.md` 생성
4. 첫 주간 계획 수립 → `study-plan.md` 생성
5. `mastery.md` 초기 버전 생성 (자가 보고 기반, 진단 전)
6. `spaced-queue.md` 초기화 (빈 큐)
7. 첫 진단 세션 (가장 불안한 주제로 /diagnose → 자가 보고 vs 실제 비교 → mastery 갱신)

**감지 조건:** `goals.md`, `study-plan.md`, `mastery.md` 중 하나라도 없으면 Cold Start 트리거.
